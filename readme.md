Contexte

Dans le cadre du Bloc 2 en Génie Logiciel (L1 IFRI), notre groupe a reçu le code source d'une mini-application web à analyser.
Cette application a été développée par un ancien stagiaire fictif de DevStart Agency. Le code fonctionne partiellement, comporte peu ou pas de documentation, et présente plusieurs problèmes de lisibilité.
Notre mission est de comprendre, analyser et documenter ce code, sans le réécrire.


Objectifs du projet

- Lire et comprendre un code écrit par une autre personne
- Identifier le rôle des fonctions, variables et structures
- Détecter les problèmes de lisibilité ou d’organisation
- Proposer des pistes d’amélioration
- Rédiger une documentation claire et utile
- Collaborer efficacement via GitHub

Structure du dépôt

├── original/ # Code source original non modifié

├── annotated/ # Code commenté et expliqué

├── rapport/ # Rapport final en PDF

└── README.md # Présentation du projet


Méthodologie adoptée

Phase 1 : Exploration du code

- Lecture complète du projet
- Observation du fonctionnement général
- Identification des fichiers importants
- Compréhension progressive des fonctions principales

Phase 2 : Analyse critique

- Recherche des problèmes de lisibilité
- Étude de la logique utilisée
- Détection des fonctions trop longues ou confuses
- Proposition de meilleurs noms pour certaines variables/fonctions

Phase 3 : Documentation finale

- Annotation du code
- Rédaction du rapport d’analyse
- Préparation de la présentation orale

Outils utilisés

- GitHub

- Visual Studio Code

- Navigateur Web

- WhatsApp / Discord (communication)

- Outils bureautiques pour le rapport

Exemples de problèmes identifiés

- Variables mal nommées

- Manque de commentaires

- Fonctions avec trop de responsabilités

- Structure générale peu claire

- Répétitions inutiles dans le code

Propositions d’amélioration

- Renommer les variables de façon explicite

- Découper les fonctions trop longues

- Ajouter des commentaires utiles

- Mieux organiser les fichiers

- Séparer logique métier et affichage

Utilisation du projet

1. Télécharger ou cloner le dépôt GitHub

2. Ouvrir le dossier "original/"

3. Lancer le fichier principal ("index.html" ou autre fichier de départ)

4. Consulter le dossier "annotated/" pour la version commentée


Conclusion
Ce projet nous a permis de comprendre qu’en développement logiciel, il ne suffit pas de savoir coder. Il faut aussi savoir lire, analyser, expliquer et améliorer du code existant. Cette compétence est essentielle dans le monde professionnel.
